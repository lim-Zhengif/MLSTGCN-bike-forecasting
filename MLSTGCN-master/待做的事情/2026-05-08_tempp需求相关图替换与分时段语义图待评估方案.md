## 1. 关联论文与启发

参考论文：

`e:\学校\2026年实验后找能够引用的论文\！-可当作解释--为什么使用站点相似比较Predicting short-Term bike-Sharing demand at station level A multi-Task dynamic graph-based spatiotemporal approach\1-s2.0-S0950705125020246-main.pdf`

该论文强调：

- 站点之间的关系不能只依赖地理距离或功能相似性。
- 历史需求模式相似的站点，也应该被视为语义相邻站点。
- 站点间需求相似关系具有时间变化性，即早高峰、晚高峰、夜间、工作日、周末可能对应不同的站点相似结构。
- 骑入与骑出并不是完全独立任务，二者在时空上存在相互影响。

这与当前项目中的 `tempp_bike.npy` 需求相关图高度相关，可以作为“为什么使用站点相似比较/需求相关图”的论文依据。

## 2. 当前 `tempp_bike.npy` 的真实含义

当前项目中，`tempp_bike.npy` 对应源文件：

```text
graph_demand_correlation.npy
```

在 Top150/Top300 图构建脚本中，其生成逻辑为：

```text
每个站点的 daily_netflow 序列
    -> 站点两两 Pearson 相关系数
    -> NaN/Inf 置 0
    -> 负相关裁剪为 0
    -> 对角线设为 1
    -> 保存为 graph_demand_correlation.npy
    -> 准备训练数据时保存/复制为 tempp_bike.npy
```

因此，当前 `tempp_bike.npy` 表示的是：

```text
基于日净流量 daily_netflow 的静态 Pearson 需求相关图
```

它不是小时级语义图，也不是骑入/骑出分别建图，也不是输入相关的动态邻接矩阵。

当前最新 Top150 hist336 训练图目录中的 `tempp_bike.npy` 大致特征：

```text
shape: 150 x 150
value range: [0, 1]
symmetric: True
diagonal in file: 1
off-diagonal positive ratio: about 52.2%
off-diagonal mean: about 0.0626
```

训练时通常启用：

```text
--tempp_diag_zero true
```

因此文件内对角线虽为 1，但模型加载后会把 `A_tempp` 的对角线置 0。

## 3. 为什么不能简单说“再做 Pearson 图”

因为当前 `tempp_bike.npy` 已经是 Pearson 图，只不过它是：

```text
日净流量 daily_netflow Pearson
```

如果后续新建 `tempp_bike_pearson.npy`，应避免重复当前图。更有意义的 Pearson 候选应为：

- 小时级净流量 Pearson
- 小时骑出量 Pearson
- 小时骑入量 Pearson
- 骑入+骑出联合 Pearson
- 仅用训练期，例如 2025 年，计算的 Pearson
- 按早高峰/晚高峰/夜间/工作日/周末分组计算的 Pearson

其中“仅用训练期计算”尤其重要，可以避免使用测试期信息造成数据泄漏。

## 4. 保持模型结构不变的替换方案

当前模型加载图时固定读取五个文件：

```text
dist.npy
neigh.npy
func.npy
bike_heuristic.npy
tempp_bike.npy
```

因此，候选语义图不是直接新增第六张图，而是替换 `tempp_bike.npy` 这一位置。

推荐做法是为每个候选图复制一个独立图目录，而不是覆盖原目录。

示例：

```text
原始图目录：
data/graph/bike_hourly_safe_inventory_top150_nyc_full_predlen6_anchors_00_06_12_16_20_train2025_hist336

候选图目录：
data/graph/bike_hourly_safe_inventory_top150_nyc_full_predlen6_anchors_00_06_12_16_20_train2025_hist336_tempp_hourly_pearson
data/graph/bike_hourly_safe_inventory_top150_nyc_full_predlen6_anchors_00_06_12_16_20_train2025_hist336_tempp_dtw
data/graph/bike_hourly_safe_inventory_top150_nyc_full_predlen6_anchors_00_06_12_16_20_train2025_hist336_tempp_mix
```

每个候选目录中保留：

```text
dist.npy
neigh.npy
func.npy
bike_heuristic.npy
selected_node_mapping.csv
```

只替换：

```text
tempp_bike.npy
```

训练时保持 `data_dir` 不变，仅切换 `graph_dir`：

```text
--data_dir data/temporal_data/原数据集
--graph_dir data/graph/候选图目录
```

这样模型结构、输入特征、损失函数、训练脚本都不需要变化。

## 5. 候选语义图设计

### 5.1 hourly Pearson 图

目的：

检验小时级需求相似性是否优于当前日净流量相似性。

可选构造方式：

```text
A_hourly_net = corr(站点小时净流量序列)
A_hourly_out = corr(站点小时骑出量序列)
A_hourly_in = corr(站点小时骑入量序列)
A_hourly_joint = 平均或拼接后计算骑入+骑出联合相关
```

推荐优先级：

```text
先做 A_hourly_joint 或 A_hourly_net
```

原因是它们与当前调度目标更接近，也较容易解释。

### 5.2 DTW 图

目的：

Pearson 更关注线性同步变化，DTW 更适合捕捉“形状相似但有时间错位”的需求模式。

可选构造方式：

```text
对每个站点构造小时需求序列或典型日 24 小时需求曲线
计算站点两两 DTW 距离
将距离转换为相似度
归一化到 [0, 1]
对角线设为 1
```

相似度转换示例：

```text
sim_ij = exp(-dtw_ij / scale)
```

注意：

DTW 两两计算成本较高。Top150 可以接受，Top300 需要控制序列长度或先使用典型日曲线。

### 5.3 Pearson + DTW mix 图

目的：

综合 Pearson 的同步相关性和 DTW 的形状相似性。

构造方式：

```text
A_mix = alpha * A_pearson + (1 - alpha) * A_dtw
```

推荐先取：

```text
alpha = 0.5
```

后续如果效果明显，再做：

```text
alpha = 0.25, 0.5, 0.75
```

## 6. 分时段语义图的具体思路

当前 `tempp_bike.npy` 是一张全局静态图：

```text
所有预测时段共用同一张站点相似图
```

分时段语义图的想法是：

```text
不同时间段使用不同历史样本计算站点相似关系
```

例如：

```text
A_morning: 只用 07:00-10:00 历史需求计算
A_evening: 只用 16:00-20:00 历史需求计算
A_night: 只用 00:00-05:00 历史需求计算
A_weekday: 只用工作日历史需求计算
A_weekend: 只用周末历史需求计算
```

直观解释：

- 早高峰：住宅区出行站点之间可能更相似，办公区到达站点之间可能更相似。
- 晚高峰：办公区流出、住宅区流入，站点相似关系可能反转。
- 夜间：整体需求较低，交通枢纽和娱乐场所附近站点可能表现不同。
- 周末：公园、商圈、景点、交通枢纽的需求模式可能更接近。

因此，站点的地理位置不变，但“需求模式上的相似关系”会随时间段变化。

## 7. 分阶段实施路线

### 阶段 A：不改模型，只做静态替换实验

这是最低风险路线。

实验组：

```text
baseline_current_tempp: 当前日净流量 Pearson 图
hourly_pearson_tempp: 小时级 Pearson 图
dtw_tempp: DTW 图
mix_tempp: Pearson + DTW 混合图
```

做法：

```text
每个候选图复制一个 graph_dir
把候选图保存为该目录下的 tempp_bike.npy
训练时只切换 --graph_dir
```

优点：

- 不改模型结构。
- 不改数据集结构。
- 不影响已有训练结果。
- 易做消融实验。

### 阶段 B：仍不改模型，先验证分时段图是否有信号

先分别构造：

```text
tempp_morning.npy
tempp_evening.npy
tempp_night.npy
tempp_weekday.npy
tempp_weekend.npy
```

再用“静态替换”的方式分别训练：

```text
只用 morning 图替换 tempp_bike.npy
只用 evening 图替换 tempp_bike.npy
只用 weekday 图替换 tempp_bike.npy
```

这一步虽然还没有做到按样本动态切图，但可以回答一个关键问题：

```text
分时段构造出来的语义图是否比全局图更有预测价值？
```

如果没有明显收益，则暂时不值得大改模型。

### 阶段 C：改模型支持按 anchor_hour 选择语义图

这是较大改动，不建议第一步做。

大致需要：

```text
1. 数据集返回每个样本的 anchor_hour 或 sample_datetime
2. BikeGraph 同时加载多张 tempp 图
3. FusionGraphModel.forward 接收时间条件
4. MSTGCN.forward 将时间条件传给 fusiongraph
5. 根据 anchor_hour 选择 A_morning/A_evening/A_night
6. 再与 dist/neigh/func/bike_heuristic 做五图融合
```

风险：

- 需要改数据流。
- 旧 checkpoint 不一定兼容。
- 训练和评估脚本都要同步调整。
- 如果阶段 A/B 没有收益，直接做阶段 C 成本偏高。

## 10. 暂不实施原因

当前阶段先记录，不立即修改代码，原因如下：

- 当前 `tempp_bike.npy` 已经是 Pearson 需求相关图，需要先避免重复设计。
- 静态替换实验虽然风险低，但仍需要重新训练多组模型。
- 分时段动态图涉及数据集、图加载、模型 forward 和评估流程改动，属于中等以上改造。
- 应先通过静态候选图消融确认“需求语义图改造”确实有收益，再决定是否进入动态版本。

## 11. 建议后续优先级

推荐顺序：

```text
1. 保留当前 baseline_current_tempp
2. 构造 hourly_joint_pearson tempp
3. 构造 dtw_tempp
4. 构造 mix_tempp
5. 使用相同训练配置做 graph_dir 替换实验
6. 如果候选图优于当前图，再做 morning/evening/weekend 分时段图
7. 如果分时段图仍有收益，再考虑模型级动态选图
```

最保守的第一步：

```text
只做 hourly_joint_pearson_tempp 与当前 tempp 的对比
```

这一步最容易解释，也最符合“保持模型结构不变、只替换需求相关图”的原则。
