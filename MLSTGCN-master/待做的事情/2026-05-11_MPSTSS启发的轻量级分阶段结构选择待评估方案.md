## 3. 需要明确的边界

轻量级自动搜索本质上不是完整 NAS。

它更接近：

```text
按照预设参数组合自动训练
    -> 记录验证集/测试集指标
    -> 汇总结果
    -> 选择表现最好的配置
```

因此，论文中不宜表述为：

```text
本文提出了一种 NAS 方法
```

更稳妥的表述是：

```text
受 MPSTSS 分阶段搜索思想启发，本文采用轻量级分阶段结构选择策略，对空间图融合配置与时间建模配置进行系统筛选。
```

或：

```text
stepwise hyperparameter/model-component selection
```

## 4. 为什么已有 hist_len=72/168/336 仍有价值

当前已经完成的：

```text
hist_len = 72 / 168 / 336
```

可以视为时间窗口长度消融实验，也可以视为轻量搜索中的一个子步骤。

它回答的是：

```text
历史输入窗口长度取多少更合适？
```

但它尚未回答：

```text
空间图融合配置与时间窗口配置组合起来，哪一组最优？
```

例如可能出现：

```text
hist336 + graph_topk20 效果较好
hist168 + graph_topk30 可能更好
hist72 + 更强时间模块可能也有竞争力
```

因此，如果后续要做轻量分阶段搜索，已有 hist_len 实验不需要废弃，而是可以作为时间阶段的重要依据。

## 5. 推荐的 stepwise search 设计

### 阶段 A：空间图建模配置选择

固定时间结构，例如：

```text
hist_len = 168 或 336
pred_len = 6
time_kernel_size = 3
nb_block = 当前默认值
```

只搜索空间相关参数：

```text
graph_topk = 10 / 20 / 30
graph_attention = true / false
matrix_weight = true / false
graph_sparsify_mode = topk / row_mean / topk_or_row_mean
tempp 图版本 = current / hourly_pearson / dtw / mix
```

目标：

```text
找到更适合当前共享单车数据的图融合配置。
```

这一步最贴合当前模型创新点，因为当前模型核心是多图融合。

### 阶段 B：时间建模配置选择

固定阶段 A 得到的较优空间配置，再搜索时间相关参数：

```text
hist_len = 72 / 168 / 336
time_kernel_size = 3 / 5
nb_block = 2 / 3
cheb_k = 2 / 3
channel_attention = true / false
```

目标：

```text
找到更合适的历史窗口和时序建模强度。
```

已有的 hist_len=72/168/336 实验可以纳入这一阶段。

### 阶段 C：最终重训与独立测试

使用阶段 A 和阶段 B 选出的最佳配置，进行正式重训：

```text
固定随机种子
使用完整训练预算
记录 best checkpoint
在 Feb 2026 holdout 上测试
与 baseline / GRU / naive 等结果对比
```

## 6. 最小实现方式

不需要改模型结构。

可以新增一个实验调度脚本，负责自动生成训练命令，例如：

```text
run_stepwise_search.py
```

脚本职责：

```text
1. 定义候选参数网格
2. 逐组调用 train_bike.py
3. 为每组实验生成独立 project name
4. 读取 logs/.../metrics.csv
5. 汇总 val_mae / test_mae / horizon_mae 等指标
6. 输出 stepwise_search_results.csv
```

也可以暂时不用脚本，先手工跑少量核心组合，但论文中最好统一整理成表格。

## 7. 推荐的低成本搜索空间

为了避免实验量过大，建议第一版不要做全组合暴力搜索。

空间阶段优先搜索：

```text
S0: 当前配置
S1: graph_topk = 10
S2: graph_topk = 20
S3: graph_topk = 30
S4: graph_sparsify_mode = topk_or_row_mean
S5: graph_attention = false
```

时间阶段优先搜索：

```text
T0: hist_len = 72
T1: hist_len = 168
T2: hist_len = 336
T3: time_kernel_size = 5
T4: nb_block = 3
```

这样可以先把实验数量控制在 10 组以内。

如果后续确实有效，再扩展更完整的组合。

## 8. 与完整 NAS 的区别

完整 NAS 通常包括：

```text
搜索空间定义
架构参数
权重参数
搜索阶段
重训练阶段
可能还包括可微搜索或强化学习/进化搜索
```

本方案不做这些复杂部分。

本方案只做：

```text
有限候选配置
分阶段训练比较
结果表格汇总
最佳配置重训
```

因此，它更适合作为毕业设计中的工程化、可复现模型选择策略。

```text
受 MPSTSS 中分阶段结构搜索思想启发，本文未直接采用高成本的完整神经架构搜索，而是结合共享单车需求预测任务特点，设计了轻量级分阶段结构选择策略。
```

```text
首先，在固定时间建模配置的条件下，对多图融合相关参数进行筛选，以确定更适合站点级需求传播的空间结构；随后，在固定空间配置的基础上，对历史窗口长度和时间卷积结构进行比较，从而降低联合搜索成本并增强实验可解释性。
```

```text
该策略本质上是一种面向模型组件和关键超参数的 stepwise selection，而非完整 NAS，因此具有较低工程成本和较高复现性。
```

如果只保留 hist_len 实验，也可以写：

```text
为分析不同历史周期长度对小时级共享单车需求预测的影响，本文设置 72、168 和 336 小时三种历史窗口进行对比，以验证短期波动、周周期信息及更长历史上下文对预测性能的影响。
```

## 10. 暂不实施原因

当前阶段先记录，不立即执行，原因如下：

- 轻量搜索本质上仍需要多次训练，计算成本不低。
- 当前已有 hist_len=72/168/336 实验，可先作为时间窗口消融使用。
- 若没有足够实验预算，不宜扩大参数搜索范围。
- 该方案更适合作为后续增强论文实验系统性的补充，而不是当前模型主改动。

## 11. 建议后续优先级

推荐顺序：

```text
1. 先把已有 hist_len=72/168/336 整理成时间窗口消融表。
2. 如果需要增强“模型选择依据”，补做少量空间参数实验。
3. 将空间实验和时间窗口实验组织成 stepwise selection。
4. 仅当实验结果明显优于 baseline，再在论文中重点展开。
```

最保守的后续补充实验：

```text
固定 hist_len=336
只比较 graph_topk = 10 / 20 / 30
```

如果这一组结果有明显差异，再考虑扩大到更多空间和时间组合。
