# ASTN G-MSA 分组多头自注意力待评估方案

记录日期：2026-05-07

状态：待评估，暂不实施代码改动。

## 参考论文

论文：`An Adaptive Spatial-Temporal Method Capturing for Short-Term Bike-Sharing Prediction`

本地路径：

`E:\学校\2026年实验后找能够引用的论文\An_Adaptive_Spatial-Temporal_Method_Capturing_for_Short-Term_Bike-Sharing_Prediction\An_Adaptive_Spatial-Temporal_Method_Capturing_for_Short-Term_Bike-Sharing_Prediction.pdf`

核心可借鉴点：论文提出 Group Multi-head Self-Attention，即 G-MSA。它将所有站点划分为小组或 patch，在组内做多头自注意力，用于同时捕捉局部空间关系和潜在远距离相似关系。

## 与当前项目的关系

当前项目主线是共享单车小时级骑入/骑出预测、安全库存区间与调度应用。

当前模型的核心结构是：

1. 多个先验图输入 FusionGraph。
2. `fusiongraph.py` 中通过 spatial attention、graph attention 和 gated fusion 学习融合邻接矩阵。
3. 融合图输入 MSTGCN，完成时空预测。

ASTN 的 G-MSA 与我们模型的关系：

- ASTN 的 G-MSA 主要对“站点节点”做分组自注意力。
- 我们当前的 FusionGraph 注意力主要服务于“五张先验图的融合”。
- 二者不完全相同，但可以结合：把 G-MSA 的“分组站点注意力”思想加入当前 `spatialAttention`，作为空间注意力的一个可开关变体。

## 初步判断

可以引用，也可以尝试改造进模型。

但不建议直接替换当前 FusionGraph 注意力结构。更稳妥的方式是新增一个可控实验分支：

`base FusionGraph attention -> grouped spatial attention variant`

这样可以保留现有模型主线，并通过消融实验验证分组注意力是否真的有效。

## 建议的最小改造方案

目标：在不大幅重写模型的前提下，将 ASTN 的分组多头自注意力思想引入 FusionGraph。

建议位置：

- `版本管理/2026-04-09_小时级骑入骑出_安全库存区间/models/fusiongraph.py`
- 必要时同步根目录 `models/fusiongraph.py`
- 训练入口使用 `train_bike.py` 暴露开关参数

建议新增配置参数：

```text
--fusion_attention_variant base/grouped
--group_attention_mode pearson/shuffle/both
--group_size 16
--group_seed 0
--group_mask_weight 1.0
```

其中：

- `base`：保持当前注意力机制不变。
- `grouped`：启用分组空间注意力。
- `pearson`：基于历史需求相关性或 `tempp_bike.npy` 进行分组，对应论文 pg-MSA 思路。
- `shuffle`：固定随机种子打乱站点后分组，对应论文 sg-MSA 思路。
- `both`：同时使用 pearson group 和 shuffle group。

## 可实现的技术路线

### 方案 A：注意力 mask 方案，优先推荐

在 `spatialAttention.forward()` 中，原逻辑会计算：

```text
attention = softmax(QK^T / sqrt(d))
```

可以在 softmax 前加入 group mask：

```text
attention_logits = QK^T / sqrt(d)
attention_logits = attention_logits + group_bias
attention = softmax(attention_logits)
```

其中 `group_bias` 让同组站点更容易互相关注，非同组站点不完全屏蔽，只降低权重。

优点：

- 改动小。
- 不破坏当前 FusionGraph 主流程。
- 可通过参数关闭。
- 适合做消融实验。

风险：

- 如果 mask 过强，可能损害多图融合原本的全局建模能力。

### 方案 B：双分支 grouped attention 方案

在当前 `spatialAttention` 外新增分支：

```text
H_base = current spatial attention
H_group = grouped spatial attention
H = gate(H_base, H_group)
```

优点：

- 更接近“增强模块”而不是硬约束。
- 可以学习何时相信原注意力，何时相信分组注意力。

风险：

- 参数更多，训练成本更高。
- 论文写作上更好讲，但实现和调参成本也更大。

### 方案 C：直接替换为 ASTN G-MSA，不推荐

把 ASTN 的 G-MSA 整块替换进当前模型。

不推荐原因：

- ASTN 是短时预测框架，我们当前模型是多图融合 + MSTGCN。
- 直接替换会让模型主线偏离“动态多图融合”。
- 不利于判断性能提升来自 G-MSA 还是整体结构变化。

## 分组来源设计

### Pearson 分组

可直接使用现有图：

- `tempp_bike.npy`
- 或 Top150/Top300 数据集中的 `tempp_bike.npy`

含义：历史骑入/骑出模式相似的站点分到同组。

这与论文 pg-MSA 思路一致。

### Shuffle 分组

使用固定随机种子对站点编号打乱，再按 `group_size` 切分。

含义：打破地理距离限制，让远距离站点有机会进入同一组。

这与论文 sg-MSA 思路一致。

### 可选：混合分组

分别计算 pearson-group attention 与 shuffle-group attention，再用平均或门控融合。

## 建议实验顺序

优先使用当前较成熟的 Top150 rolling anchor 实验链路：

- 数据：`bike_hourly_safe_inventory_top150_nyc_full_predlen6_anchors_00_06_12_16_20_train2025_hist168`
- 预测长度：`pred_len=6`
- 历史长度：`hist_len=168`
- 现有较优配置：`lr=5e-5`、`huber_delta=2.0`、`graph_topk=20`

建议消融：

1. `base`：当前模型，不启用分组注意力。
2. `grouped_pearson`：只启用 Pearson 分组。
3. `grouped_shuffle`：只启用 Shuffle 分组。
4. `grouped_both`：同时启用两类分组。

评价指标：

- `val_mae_epoch`
- holdout MAE/RMSE
- 按 horizon 的 MAE
- 按站点活跃度分组的 MAE
- 高峰小时风险识别指标
- 后续可接调度仿真，看是否降低缺车/满桩风险

## 论文写作可用表述

可以把这篇论文作为“分组空间注意力”的依据引用。

建议表述方向：

ASTN 通过 G-MSA 将共享单车站点划分为小组，并在组内执行多头自注意力，以兼顾局部空间依赖和潜在远距离相似关系。受此启发，本研究考虑在动态多图融合模块中引入分组空间注意力约束，使模型在融合距离图、OD 图、POI 语义图与需求相关图时，更有针对性地关注模式相似或随机扰动形成的站点子集。

注意：如果后续没有实际实现和实验验证，只建议把它放在相关工作或未来工作中，不要写成本文已采用的方法。

## 当前不立即实施的原因

1. 当前模型已有 `tempp` 需求相关图，Pearson 分组可能与现有图信息重复。
2. 分组注意力会改变 FusionGraph 的空间注意力权重分布，需要消融验证。
3. 这属于模型结构改造，应单独开实验版本，避免和当前 Top150/Top300 主线结果混在一起。

## 后续触发条件

满足以下任一条件时，可以考虑正式实现：

1. 当前模型在低活跃时段、夜间或稀疏站点上误差明显偏高。
2. 站点级误差分析显示相似功能区或相似需求模式站点没有被充分利用。
3. 需要为论文增加一个明确的模型结构创新点，而不仅是数据和调度应用创新。
4. 已准备好运行至少四组消融实验。
